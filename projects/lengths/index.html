Lengths, in the context of CSS, can be broken down in to two categories, Absolute and Relative.

Absolute Length Units

Absolute length units are probably the most familiar to those just starting out. An absolute length is an static measurement not defined by another element or the viewport. Like inches or feet in the real world, absolute values within CSS operate much the same way.

px

The first and most common absolute element within our CSS is the pixel length. Before I say any more on pixel, it is important to first remember that pixel, in terms of CSS lengths, has nothing to do with screen pixels, and therefore should be thought of as device pixels.

Notated as px in CSS, pixels are a unit of measure that are great for setting fixed font sizes, heights, widths, borders, letter spacing and media queries to our elements. These values will not change when the browser window is resized.

Aa
font-size: 30px
Aa
font-size: 50px
height/width: 40px
height/width: 75px

So that's it? Just pixels?

Pixels are not the only absolute length value, but they are by far the most common. Other absolute units include points (pt), millimeters (mm), centimeters (cm) and inches (in), but none of these have any advantage to pixel and are not commonly used.

Relative Length Units

Relative length units are font-relative. This means that the actual length is determined by the font-size of the element. Relative length units are great because the variable in sizes allows us to create smooth, scalable layouts. This is extremely important in responsive web design.

There are numerous relative length values. I have highlighted a few them below. I have also provided a few instances when I use them.

em

An em's size is determined from the inherited font-size of its parent element. With em we are not limited to strictly defining font sizes, but it is also common practice to use em to define margins, padding and line-heights as well.

Remember, if a font size is not defined, the em unit's size will inherit the base font size of its nearest parent element that has a defined font size and calculate from there.

Aa
font-size: 1em
Aa
font-size: 4em
box
padding: 5px
box
padding: 1em

How is em size calculated?

A very simple explanation that really helped put how em work goes like this. Assuming there is no outside influencing CSS, let's say we have an element with a font-size of 20px and a width of 5em. Knowing that one em is equal to the defined font-size of 20px, we can determine the width of the element in pixels. The element's width is actually 100px (20px*5em=100px). Still don't understand? Click here for an example.

rem

Root em units are notated as rem in our CSS. These units are extremely similar to em units in the sense that its size is actually dictated from another element's defined font-size.

In the case of rem, the size is defined from the root element, which is usually the html element within our CSS

Aa
font-size: 2.8rem
Aa
font-size: 2.8em

Wait, I'm confused...

You can see that the text with a font size of 2rem is slightly smaller than the text with a font-size of 2em. This is because the root (html) element font size is set to 100%, while the body font size is set to 125%.

Remember what I said earlier? If a font size is not defined, the em unit's size will inherit the base font size of its nearest parent element that has a defined font size. In this case, the body is the nearest parent element.

%

Percentages are notated with a % in CSS and are an extremely common property. Although not technically a length unit, percentages have a wide variety of uses and are most closely related to the relative length units.

Percentages are great for creating fluid web pages that change and adjust given the user's viewport. This helps tremendously when talking about responsive web design.

width: 40px
width: 90%

Go ahead and test for yourself!

If you resize your window you will notice that the box with a width of 40px will not grow or shrink, while the box with a width of 90% will adjust with your viewport horizontally.

I personally like using percentages for containers and various aspects of a page's layout. These boxes and containers that contain content can easily adjust to perfectly fit the user's viewport whichever device they are using.
