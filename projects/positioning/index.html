Positioning
A brief introduction to the web's most common positioning properties.

When thinking about positioning in terms of HTML and CSS, most of us agree that there are multiple ways to accomplish the same goal. What we don't always know is which is the most efficient.

In our last blog we talked about the advantages as well as the disadvantages of floats. Floats are an extremely useful tool that really give us a lot of freedom to organize and put things where we want.

Let's add to what we already know about positioning from floats and talk about a few very common positioning types. We will also talk about how these properties work with one another and at the end we will take a look at another very essential property that coincides with positioning.

Let's take a look at some of the most common positioning properties that we currently have available to us.

  • 1. Static

    This property is the default positioning property for all elements. A static property keeps your element within the normal flow of your HTML document and does not work with any offset properties like top, right, left and bottom.

    One of the most common uses for static is that it gives us the ability to forcefully remove a positioning property from an element that was inherited elsewhere.

  • 2. Relative

    Relative positioning is one of the most used and most confusing of the positioning properties. A relative property works very similar to static in the sense that it will not take your element out of the normal flow of your HTML document. The big difference between static and relative is that with relative you are able to use offset properties.

    It is important to remember that changing an element's position to relative and not assigning offset properties will result in no change to the element.

  • 3. Absolute

    Elements with an absolute positioning property are taken out of the normal flow of your HTML document and are, by default, attached to the closest relative or absolute positioned parent element. With absolute positioning you can move your element wherever you want on your page using offset properties.

    The only catch to absolute is if not attached to another element, it is at the mercy of the user's viewport because it has been taken out of the normal flow of the HTML document.

  • 4. Fixed

    An element with a fixed position is essentially anchored to a certain spot within the user's viewport. A fixed position is neat because the element will remain in its location even when the user scrolls up, down or resizes the window. This type of positioning property is not common, but certainly has some uses. A Navigation you want to follow you down the page is a great example.

Child

Bowser will represent the child element. The child element is contained by its parent element.

Child: Static

Uh oh, it looks like Mario is stuck behind Bowser. This is because when a child element is put inside of a parent element, the child element is essentially layered on top. In this example, both the parent and child elements are set to static. A static positioning property is the default of every element we use.

Parent: Static
Child: Relative

Here Mario is now visible, but things still look off. Our parent element, Mario, is positioned as static and Bowser is now positioned relative. In addition to changing the positioning, we have also added offeset properties of left 100px and bottom 50px. Offset properties give us control of exactly where we want to position elements these elements.

Parent: Relative
Child: Absolute

Here we have changed our parent element to relative and added offset properties bottom -25px and left -50px. We then set our child element, Bowser, to absolute with an offset property of left 130px. This ensures that the child element will be 130px away of wherever our Mario is moved.

What about the Star?

The star that has been following us on the left side of the screen is set to a fixed position. This property allows whatever element we specify to be set to a fixed area within the browser's viewing area, or viewport.

z-index

The z-index property controls the stacking order of positioned elements, allowing them to overlap one another. An easy way to think of it is like using layers. How z-index works is very simple, basically, the higher the number associated with z-index means the greater priority for it to be the top-most layer.

In this situation the star will always be behind Luigi in the stack order since it's z-index is 1 and Luigi's z-index is 2.

On a fun, side note, the z-index number range is huge, we're talking billions. The current max range supported by most browsers is 2,147,483,638.

Changing the z-index of Luigi to 1, and the star to 2 reverses the stack order.

This allows the star to come forward where it now rests comfortably in Luigi's hand.

When using z-index it is important to remember is that there are boundaries. The first thing to remember is that z-index only works on positioned elements.

The second thing you want to take into account is how your elements are nested within each other. When nested, your child element can never have a greater z-index than its parent.
